import java.util.concurrent.ConcurrentHashMap;
import java.util.concurrent.locks.StampedLock;

public class TransactionManager {
    private ConcurrentHashMap<String, Double> accounts;
    private StampedLock lock;
    private int sequenceNumber = 0;

    public TransactionManager() {
        this.accounts = new ConcurrentHashMap<>();
        this.lock = new StampedLock();
    }

    public void deposit(String accountName, double amount) {
        if (amount < 0) {
            throw new IllegalArgumentException("Deposit amount cannot be negative.");
        }
        int stamp = lock.writeLock();
        try {
            sequenceNumber++;
            accounts.put(accountName, accounts.getOrDefault(accountName, 0.0) + amount);
        } finally {
            lock.unlockWrite(stamp);
            System.out.println(String.format("%s deposited %f. New balance: %.2f", accountName, amount, accounts.get(accountName)));
        }
    }

    public void withdraw(String accountName, double amount) {
        if (amount < 0 || amount > accounts.getOrDefault(accountName, 0.0)) {
            throw new IllegalArgumentException("Withdrawal amount cannot be negative or exceed current balance.");
        }
        int stamp = lock.readLock();
        try {
            sequenceNumber++;
            accounts.put(accountName, accounts.getOrDefault(accountName, 0.0) - amount);
        } finally {
            lock.unlockRead(stamp);
            System.out.println(String.format("%s withdrew %f. New balance: %.2f", accountName, amount, accounts.get(accountName)));
        }
    }

    public double getBalance(String accountName) {
        int stamp = lock.readLock();
        try {
            return accounts.getOrDefault(accountName, 0.0);
        } finally {
            lock.unlockRead(stamp);
        }
    }
}

1. We use a `ConcurrentHashMap` to store the accounts and balances.
2. A `StampedLock` is used as a replacement for a traditional lock. This allows us to acquire multiple locks (read and write) in sequence, which helps improve performance by allowing readers to continue executing concurrently.

To understand how this works:

- When we acquire the write lock (`lock.writeLock()`), we increment the sequence number (`sequenceNumber++`) before updating the account balance.
- After updating the balance, we release the write lock using `unlockWrite()`. If another thread is waiting for a read lock or if the read lock has expired, it will automatically be granted.

The benefits of this approach include:

- Simultaneous reads are possible because readers do not block each other. Instead, they simply increment and return their result.
- Writes still have exclusive access to update balances in case concurrent reads interfere with updates.
